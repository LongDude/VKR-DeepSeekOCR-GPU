# Что это?
Тестовый стенд для модели распознания документов DeepseekOCR. С помощью утилиты make в автоматизированном режиме развертывается на
хосте и автоматически обрабатывает PNG и PDF файлы из raw_files (.png/.pdf) и сохраняет текстовое содержимое в директорию processed_files в формате Markdown (.md). 

Целью разработки тестового стенда были:
- Обкатка и тестирование предварительно собранного базового слоя контейнера longdude/transformers-base:5060ti с устанвленным стеком библиотек и инструментов для быстрой развертки ИИ на совместимых устройствах;
- Наработка навыка автоматического развертывания контейнеров с нестандартной средой выполнения;

# Руководство пользования
Управление проектом осуществляется с помощью набора пользовательских скриптов в ./cmd.
## Предварительные требования
- Установленный Python3.11 для выполнения на хосте. Проверяется командой `python3.11 -V` или `py -3.11 -V`
- Драйвера Nvidia 575.x или новее. Проверяется командой `nvidia-smi`
- Набор инструментов Nvidia CUDA 12.8 или новее. Проверяется командой `nvcc -V`

## Установка
Склонировать репозиторий в свободную папку.
```
git clone https://github.com/LongDude/VKR-DeepSeekOCR-GPU
```

Для запуска в изолированной среде (контейнере) необходимо собрать контейнер и проинициализировать виртуальную среду для него:
```
cd /cmd
./build_image.sh # Сборка базового образа Ubuntu 24.02 + Cuda 12.8 + Python 3.11
./init-container-venv.sh # Создание venv и установка всех необходимых библиотек для Python
```

Для разработки приложения IDE необходим доступ к виртуальной среде. Все инициализированные виртуальные среды расположены в папке venv. 
Для Linux при создании контейнера автоматически создается виртуальная среда .venv-lin; также её можно вручную проинициализировать/обновить с помощью утилиты `/cmd/init-venv.sh`
```
cd /cmd
./init-venv.sh
```
Для Windows аналогичным образом инициализирутся виртуальная среда .venv-dev с помощью пользовательского скрипта `/cmd/init-venv.ps1` (Powershell)
```
cd /cmd
./init-venv.ps1
```

## Запуск
Для запуска в контейнере используется скрипт `run-container.sh`
```
cd /cmd
./run-container.sh
```

Для запуска контейнера в режиме командной оболочки используется скрипт /shell.sh. 
``` 
cd /cmd
./shell.sh
```

Также при использовании виртуальной среды .venv (.venv-lin или .venv-dev) можно вручную запустить программу без изоляции контейнера. Точка входа - /src/main.py

# Примение
Файл src/ModelConfig.ini дублируется и сохраняется как src/ModelConfig.local.ini. В локальную конфигурацию вносятся пользовательские настройки для отдельного экземпляра программы. 
Файлы в формате .pdf/.jpeg/.png размещаются в папке /raw_files. В зависимости от параметров программы через определенное время формируются файлы в формате Markdown (.md) с текстом программы, размещаемые в папке /processed_files
